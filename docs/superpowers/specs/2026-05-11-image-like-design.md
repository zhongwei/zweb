# Image Like (Heart) Feature Design

## Overview

Add a click-to-like (heart) toggle to images in the `g/` gallery. Users can heart/unheart individual images from both the thumbnail grid and the full-size viewer. Like state is persisted via a backend API that writes a JS file consumed by the frontend.

## Storage Format

File: `g/likes.js`

```js
var likes = [[10011,1,3,8,19],[11300,2,7,23,190]];
```

- Each sub-array: first element is the directory number, remaining elements are liked image numbers in **ascending order**.
- No entry for a directory means zero likes in that directory.
- If file does not exist or is empty, treat as `var likes = [];`.

## Backend API

Add `tiny_http` crate to `Cargo.toml`. Start a local HTTP server on `127.0.0.1:18832` inside `main.rs`, alongside the existing wry webview.

### Endpoints

**`POST /api/like`** — toggle like for a single image.

Request body (JSON):
```json
{"dir": 10011, "img": 3}
```

Response (JSON):
```json
{"liked": true}
```

Behavior:
1. Read `g/likes.js` (or treat as empty if missing).
2. Parse the `likes` array.
3. If `(dir, img)` exists → remove it, respond `{"liked": false}`.
4. If not → add it (insert `img` in ascending order), respond `{"liked": true}`.
5. Rewrite `g/likes.js` with the updated data.

**`GET /api/likes`** — return current likes (optional, frontend can also read the `likes` variable directly after script load).

Response (JSON):
```json
[[10011,1,3,8,19],[11300,2,7,23,190]]
```

## Frontend Changes (g/ directory)

### Files Modified

- `g/index.html` — add `<script src="likes.js"></script>` before `main.js`
- `g/main.js` — add like logic, API calls, UI updates
- `g/style.css` — add heart button styles

### UI Components

**1. Heart button on thumbnail cards (`.item`)**

- Position: absolute, right-bottom corner of the card.
- Default: semi-transparent white heart outline, visible on card hover.
- Liked: solid red heart, always visible.
- Click: `stopPropagation` to prevent opening the viewer, calls `POST /api/like`, updates local state.

**2. Heart button in full-size viewer (`#viewer`)**

- Position: top-left area (opposite side of close button).
- Same visual states as thumbnail heart.
- Click: toggles like, syncs with thumbnail state.

**3. Heart icon**

- Inline SVG path (no external icon library).
- Outline state: `fill="none" stroke="white"`.
- Liked state: `fill="#ff4757" stroke="#ff4757"`.

### JS Logic

```js
var likedSet = {};  // { dir: Set([img, ...]) }

// Initialize from likes.js variable
function initLikes() { ... }

// Check if liked
function isLiked(dir, img) { ... }

// Toggle via API
function toggleLike(dir, img) { fetch('POST /api/like', ...) }

// Update all heart buttons for a given (dir, img)
function updateHeartUI(dir, img, liked) { ... }
```

On page load:
1. `likes.js` is loaded (provides `var likes`).
2. `initLikes()` builds the `likedSet` lookup.
3. Hearts are rendered during `renderNextChunk()` and `renderHome()`.

On heart click:
1. `toggleLike(dir, img)` → API call.
2. Update `likedSet` locally.
3. Update heart UI immediately (optimistic update).

### Event Handling

- Heart buttons use `click` + `stopPropagation` so clicking heart does NOT open the viewer.
- In the viewer, heart click does NOT close the viewer.

## Dependencies

- Add `tiny_http` crate to `Cargo.toml` for the HTTP API server.
- No new frontend dependencies.
